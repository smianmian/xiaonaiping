# 小奶瓶 AppLaunchOS Workflow Rule

Codex 必须按以下顺序推进：

1. Founder Intake
2. Product Spec
3. MVP Scope
4. User Stories
5. Acceptance Criteria
6. UX Flow
7. UI Contract
8. Visual System
9. Client Architecture
10. Data Model
11. Sync Rules
12. Backend Decision
13. Analytics Privacy
14. Privacy & Security
15. Regional Launch Strategy
16. Implementation Plan
17. Coding
18. QA
19. Release Checklist
20. Proof Pack
21. Post Launch Operations

第一版只做 iOS 原生 App。禁止从模糊想法直接进入 Coding。禁止在后端决策前创建服务器、数据库、API 或管理后台。
