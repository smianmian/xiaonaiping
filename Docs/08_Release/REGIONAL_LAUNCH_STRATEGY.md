# REGIONAL_LAUNCH_STRATEGY.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：区域上线策略更新
- 日期：2026-05-28
- 当前结论：首发香港和美国；中国大陆不进入第一版首发

## 已确认事实

1. 小奶瓶涉及儿童、照片、家庭和成长记录。
2. 第一版免费验证。
3. 第一版不做社区、AI、订阅、电商。
4. 第一版需要账号、备份恢复和服务器存储照片原图。
5. 首发地区为香港和美国。
6. 中国大陆不进入第一版首发。
7. 疫苗模板覆盖国内 + 香港，崩溃上报进入第一版并使用 Apple 原生渠道。
8. API 服务与对象存储区域需在发布前按合规策略确认。

## 合理推断

1. 美区/港区首发能先验证真实家庭使用，避开中国大陆备案前置不确定性。
2. 后端和对象存储的数据区域应在隐私政策中清楚说明。
3. 中国大陆后续上线需要儿童/未成年人、个人信息保护、APP 备案、ICP 判断和 SDK 清单。

## 待我确认的问题

1. Apple Developer 公司账号和 D-U-N-S 是否已准备。
2. 是否已有隐私政策 URL、用户协议 URL 和支持 URL。
3. 是否需要英文和繁中元数据。
4. 小奶瓶 API 正式域名。
5. 付费 Apple Developer 账号开通时间。

## 不进入第一版的功能

1. 中国大陆首发。
2. 分地区复杂商业化。
3. 地区差异化功能包，除非后续中国大陆上线需要。

## 策略选择

| 策略 | 当前结论 | 说明 |
|---|---|---|
| US/HK First | 已确认 | 第一版首发香港和美国 |
| China Mainland First | 不进入第一版 | 需要备案、隐私、儿童数据和服务器合规准备 |
| Global Except China | 可作为扩展 | 若后续扩展更多地区，仍排除中国大陆直到合规完成 |
| Split / Feature Flag | 暂不需要 | 第一版不做复杂地区功能差异 |

## 发布要求

1. App Store Connect 仅选择香港和美国，或不选择中国大陆。
2. Review Notes 写清楚：免费、无医疗诊断、账号备份恢复、照片原图云备份、删除账号路径。
3. 隐私政策必须覆盖跨境传输、账号、照片原图存储、删除机制。
4. 如后续中国大陆上线，另开合规项目。
5. 疫苗模板文案必须写成提醒工具，不写成医疗建议。
6. 崩溃上报必须在隐私政策和 App Privacy Label 中按实际方案披露。
