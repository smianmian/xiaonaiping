# SYNC_RULES.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：同步规则更新
- 日期：2026-05-28
- 实现状态：未实现；本文定义第一版账号备份恢复约束

## 已确认事实

1. 第一版只做 iOS 原生 App。
2. 第一版先做单人记录。
3. 第一版需要账号。
4. 第一版需要备份恢复。
5. 服务器需要存储宝宝照片原图。
6. 首发地区为香港和美国。
7. 数据涉及儿童、照片、成长记录、疫苗提醒等高敏感数据。
8. 照片可以复制进 App 私有空间。
9. 疫苗模板覆盖国内 + 香港。
10. 崩溃上报进入第一版。
11. API 服务器和对象存储第一版默认按合规策略确认。

## 合理推断

1. 同步必须 offline-first：先本地保存，再尝试上传。
2. 第一版不需要多人实时协作同步。
3. 后端服务器存储应限于账号下的备份/恢复。
4. CloudKit 暂不作为第一版默认方案，因为已确认需要第一方后端服务器存储。
5. 照片原图上传必须限于用户主动加入 App 的照片，不得扫描或自动上传系统相册。

## 不进入第一版的功能

1. 家庭多人实时同步。
2. 跨平台同步。
3. Web 后台管理。
4. 第三方云相册或第三方数据平台。
5. 社交分享同步。
6. 中国大陆首发同步策略。

## 同步目标

第一版同步目标只允许是：

1. 备份：避免换机、设备损坏或卸载导致记录和照片丢失。
2. 恢复：用户在同一账号下恢复自己的宝宝记录和照片。
3. 删除：用户能删除本地和云端数据。

不允许作为广告、画像、训练、第三方分析、运营监控或人工查看用户内容用途。

## 同步方式

- [ ] 不同步，纯本地：不再满足已确认需求，但可作为隐私设置里的关闭云备份选项待评估。
- [ ] iCloud / CloudKit：暂不作为第一版默认方案。
- [x] 第一方后端服务器：第一版需要，用于账号备份恢复和照片原图存储。

## 身份与账号

账号已确认需要。开发期先用当前可用账号/开发者资源推进；正式上架前必须准备付费 Apple Developer 账号并确认正式账号能力。

推荐方向：

1. Sign in with Apple：优先推荐，适合 App Store 上线和隐私友好恢复。
2. 邮箱登录：可行但增加验证、密码、客服和隐私成本。
3. 手机号登录：不建议第一版使用，会增加地区、短信、实名和隐私复杂度。

不能用不可解释的隐式设备追踪代替用户账号。

## 冲突处理

第一版单人记录，冲突策略保持简单：

1. 同一对象以 `updatedAt` 较新的版本为准。
2. 删除优先级高于普通更新，避免用户删除后又被恢复。
3. 云端照片删除优先级高于恢复。
4. 服务器返回冲突时保留本地副本并提示重试。
5. 如果后续做多人协作，必须重写冲突策略。

## 离线策略

1. 无网络时允许完整本地记录。
2. 本地记录标记为 `pending`。
3. 本地照片先进入 App 私有空间，再加入上传队列。
4. 网络恢复后自动尝试同步。
5. 同步失败不影响本地查看和编辑。
6. 用户必须能看到大致备份状态，但不制造焦虑。

## 删除同步

1. 用户删除本地记录后，必须向服务器发送删除请求。
2. 用户删除 App 内照片后，必须删除本地原图/缩略图，并请求删除云端原图/缩略图。
3. 删除请求失败时必须可重试，并显示“云端删除待完成”一类状态。
4. 删除宝宝档案必须删除关联记录、照片文件和云端对象。
5. 删除账号必须删除账号、记录、照片对象和可识别备份数据。
6. 服务器是否保留短期恢复窗口待确认。
7. 隐私政策必须说明服务器删除的生效时间。

## 数据恢复

1. 用户通过已确认账号登录。
2. App 从服务器拉取属于该账号的宝宝档案、记录、提醒、照片元数据。
3. App 按需下载照片原图或缩略图，具体策略待实现计划确认。
4. 本地已有数据时需避免静默覆盖。
5. 恢复过程必须允许中断和重试。

## 安全要求

1. 所有网络传输必须使用 HTTPS。
2. 服务端必须鉴权，不能通过可猜 ID 访问宝宝数据或照片。
3. 服务端日志不得记录宝宝姓名、生日、照片 URL、健康记录明文。
4. 不接入第三方分析、广告或用户画像 SDK。
5. 后端 API 规格必须在实现前单独成文。
6. 照片对象存储必须使用私有桶/私有访问，不允许公开 URL 长期裸奔。

## 多设备风险

第一版的目标是备份恢复，不是多人实时协作。若用户在多台设备登录同一账号，按单人账号的最后更新时间策略处理；不承诺多人协作一致性。

## CloudKit 评估结论

CloudKit 仍是 Apple 生态内的可选方案，但当前已确认第一版使用第一方后端服务器进行账号备份恢复和照片原图存储。若后续改为 iCloud，需要更新：

1. `DATA_MODEL.md`
2. `SYNC_RULES.md`
3. `PRIVACY_REVIEW.md`
4. App Store 隐私说明
5. 用户端同步说明
