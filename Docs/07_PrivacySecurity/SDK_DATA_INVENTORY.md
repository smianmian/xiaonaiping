# SDK_DATA_INVENTORY.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：SDK 与数据清单更新
- 日期：2026-05-28
- 当前结论：第一版需要账号和云备份，但仍默认不接入第三方分析/广告 SDK

## 已确认事实

1. 第一版只做 iOS 原生 App。
2. 第一版不做广告和商业化。
3. 第一版需要账号、备份恢复和照片原图云备份。
4. 产品涉及儿童、照片、家庭和成长记录。
5. 崩溃上报进入第一版。
6. 崩溃上报使用 Apple 原生渠道。

## 合理推断

1. 优先使用 Apple 原生框架。
2. 账号推荐 Sign in with Apple，但待确认。
3. 云端照片原图优先通过自有 API 上传，不建议客户端直连云厂商 SDK。
4. 崩溃上报使用 Apple 原生 App Store Connect / TestFlight 崩溃报告；不接入第三方崩溃 SDK。

## 待我确认的问题

1. 是否通过自有 API 上传照片，还是客户端接入云存储 SDK。
2. 付费 Apple Developer 账号开通时间。
3. Apple 原生崩溃上报脱敏验证方式。

## 不进入第一版的功能

1. 广告 SDK。
2. 归因 SDK。
3. 第三方分析 SDK。
4. 热力图 SDK。
5. 第三方云相册 SDK。

## 禁止事项

1. 不得为了“以后可能有用”采集数据。
2. 不得采集宝宝照片、儿童敏感信息作为分析事件。
3. 不得让 SDK 暗中采集设备信息、剪贴板、定位、通讯录、相册。
4. 不得接入不明来源 SDK。
5. 不得将照片原图传给分析、广告、AI 训练或第三方云相册。
6. 崩溃日志不得包含宝宝内容、照片对象 key、记录明细、token 或密钥。
