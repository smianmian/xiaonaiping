# PRIVACY_REVIEW.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：AppLaunchOS v2 隐私审查更新
- 日期：2026-05-28
- 结论：账号、备份恢复和照片原图云备份已确认进入第一版；API 服务器和对象存储 按合规策略确认；实现前必须补齐账号方式、删除 SLA、隐私政策和脱敏验证

## 已确认事实

1. 产品处理宝宝照片、出生日期、喂养、睡眠、排便、身高体重、疫苗提醒等高敏感家庭数据。
2. 第一版只做 iOS 原生 App。
3. 数据本地优先。
4. 第一版需要账号。
5. 第一版需要备份恢复。
6. 服务器需要存储宝宝照片原图。
7. 照片可以复制进 App 私有空间。
8. 首发地区为香港和美国。
9. 第一版免费验证，不做广告、电商、订阅。
10. 疫苗提醒模板覆盖国内 + 香港。
11. 崩溃上报进入第一版。
12. 崩溃上报使用 Apple 原生渠道。
13. API 服务器和对象存储第一版默认按合规策略确认。

## 合理推断

1. 本 App 应避免追踪、广告、第三方分析 SDK。
2. 后端服务器存储必须有明确用户告知、隐私政策、删除机制和安全设计。
3. 照片原图云备份是第一版最高隐私风险点。
4. 疫苗提醒只能做日程提醒，不能做医疗建议。
5. App Store 隐私标签必须披露账号、用户内容、照片/视频、健康或成长相关数据。
6. 美区/港区首发仍需关注儿童隐私、PDPO、App Privacy Label 和账号删除。
7. 崩溃上报不得包含宝宝内容、照片对象 key、记录明细、账号 token 或服务器密钥。

3. 亲友公开分享。

5. 自动上传系统相册全部照片。
6. 通讯录、定位、支付权限。
7. 中国大陆首发。

## 第三方 SDK

第一版默认不允许接入广告、归因、热力图或第三方分析 SDK。账号、备份和对象存储优先通过自有 API 完成，避免在客户端直接接入不必要云厂商 SDK。

## 服务器存储隐私要求

实现任何上传前，必须补齐：

1. 账号方式。
2. 上传字段清单。
3. 照片原图对象存储区域和访问控制。
4. 是否上传缩略图。
5. 加密传输方式。
6. 服务端访问控制。
7. 服务端日志脱敏规则。
8. 用户删除云端记录和照片机制。
9. 账号删除机制。
10. 隐私政策文案。
11. App Store 隐私标签。
12. 崩溃上报脱敏策略。

## App Store 隐私标签初步判断

| 类别 | 可能需要披露 | 说明 |
|---|---:|---|
| Health and Fitness | 是 | 喂养、睡眠、成长记录可能被视为健康/成长相关 |
| Photos or Videos | 是 | 用户导入并云备份宝宝照片原图 |
| User Content | 是 | 用户备注、照片、记录 |
| Identifiers | 是 | 账号和服务器用户 ID |
| Contact Info | 待确认 | 若账号使用邮箱 |
| Diagnostics | 是 | 崩溃上报进入第一版 |
| Location | 否 | 第一版不使用 |
| Contacts | 否 | 第一版不使用 |
| Purchases | 否 | 第一版不商业化 |

## 用户删除数据机制

第一版必须提供：

1. 删除单条记录。
2. 删除单张本地和云端照片。
3. 删除宝宝档案及全部关联数据。
4. 删除账号及云端备份数据。
5. 删除失败重试状态。
6. 删除前说明影响范围，删除后给出结果反馈。

## 风险结论

### 当前可继续

可以继续做后端实现计划、API 设计、文件存储设计、隐私政策草案和 App Store 元数据。

### 当前不可做

在以下问题确认前，不允许写代码实现上传：

1. 账号方式。
2. 对象存储私有桶命名与访问控制细节。
3. 照片删除 SLA。
4. 隐私政策和 App Store 标签。
5. 服务端日志脱敏规则。

### 高风险项

1. 照片原图云端泄露。
2. 用户无法删除云端照片。
3. 疫苗提醒被表达成医疗建议。
4. 后端日志记录敏感明文。
5. 第三方 SDK 采集儿童或家庭数据。
6. 崩溃日志带出照片路径、对象 key、token 或宝宝记录内容。
