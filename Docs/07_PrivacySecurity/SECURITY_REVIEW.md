# SECURITY_REVIEW.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：安全审查更新
- 日期：2026-05-28

## 已确认事实

1. 第一版只做 iOS。
2. 第一版需要账号、备份恢复和照片原图云备份。
3. 第一版需要崩溃上报，使用 Apple 原生渠道。
4. 当前未批准第三方分析或广告 SDK。
5. 涉及儿童和家庭敏感数据。
6. API 服务与对象存储区域需按合规策略确认。

## 合理推断

1. 最大安全风险来自照片原图上传、账号、对象存储访问控制、服务器日志和删除失败。
2. 需要在实现前设计私有对象存储、鉴权下载和删除重试。

## 待我确认的问题

1. 对象存储私有桶命名和密钥管理方案。
2. Apple 原生崩溃上报脱敏验证方式。
3. 删除 SLA。

## 不进入第一版的功能

1. 管理后台。
2. 运营查询用户数据。
3. 第三方 SDK 数据共享。
4. 公开照片链接。

## 风险清单

| 风险 | 等级 | 当前处理 |
|---|---|---|
| 未告知上传宝宝数据 | 高 | 必须在隐私和备份开启说明中披露 |
| 照片原图云端泄露 | 高 | 私有对象存储、鉴权访问、日志脱敏 |
| 账号删除不可用 | 高 | 账号进入第一版，必须做删除账号 |
| 云端照片删除失败 | 高 | 删除队列、重试、用户可见状态 |
| 服务端日志泄露敏感数据 | 高 | 后端文档禁止记录敏感明文 |
| 第三方 SDK 过度采集 | 高 | 第一版默认禁用 |
| 崩溃日志泄露敏感内容 | 高 | 只允许脱敏诊断，不记录宝宝内容/对象 key/token |
| 本地误删不可恢复 | 中 | 删除二次确认 |

## 上线前必须复核

1. App Privacy Label。
2. SDK 清单。
3. 权限弹窗触发时机。
4. 本地和云端删除机制。
5. 服务器上传字段。
6. 对象存储/对象存储访问控制。
7. 账号删除路径。
8. 崩溃上报脱敏策略。
