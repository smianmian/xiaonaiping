# DATA_SAFETY.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：数据安全更新
- 日期：2026-05-28

## 已确认事实

1. 数据本地优先。
2. 照片进入 App 私有空间。
3. 第一版需要账号。
4. 第一版需要备份恢复。
5. 服务器需要存储宝宝照片原图。
6. 数据高度敏感。
7. 崩溃上报进入第一版，使用 Apple 原生渠道。
8. API 服务与对象存储区域需按合规策略确认。

## 合理推断

1. 本地删除和服务器删除都必须可验证。
2. 账号凭证应放 Keychain。
3. 所有上传必须 HTTPS。
4. 照片对象存储必须私有访问。

## 待我确认的问题

1. 服务器数据保留周期和删除 SLA。
2. Apple 原生崩溃上报脱敏验证方式。
3. 对象存储私有桶命名和密钥管理方案。

## 不进入第一版的功能

1. 第三方数据共享。
2. 广告追踪。
3. 用户画像。
4. 公开照片分享。

## 安全要求

1. 本地业务数据存储在 App 沙盒。
2. App 内照片存储在私有空间。
3. 删除照片必须清理本地文件和云端对象。
4. 所有网络传输必须 HTTPS。
5. 账号 token 必须存 Keychain。
6. 服务端日志不得包含宝宝敏感数据。
7. 对象存储不得公开访问。
8. 备份恢复必须按账号隔离。
9. 崩溃日志必须脱敏，不得包含宝宝内容、照片对象 key、记录明细、token 或密钥。
