## 已确认事实

1. 第一版首发香港和美国。
2. 第一版是宝宝成长记录工具。
3. App 面向父母/照护者，不是直接面向儿童使用。
4. 涉及儿童照片和家庭成长数据。
5. 第一版需要账号、备份恢复和照片原图云备份。
6. 第一版免费，无 IAP。
7. 疫苗模板覆盖国内 + 香港，崩溃上报进入第一版并使用 Apple 原生渠道。
8. API 服务与对象存储区域需按合规策略确认。

## 合理推断

1. 美区需要重点关注 App Privacy Label、COPPA 风险判断、账号删除和健康/医疗表述边界。
2. 港区需要关注个人资料用途、删除渠道和跨境传输说明。
3. 不应在元数据中暗示医疗诊断能力。
4. 隐私政策至少需要英文；港区如面向中文用户，建议准备繁中。

## 待我确认的问题

1. 是否准备英文隐私政策。
2. 是否需要繁中元数据和隐私政策。
3. 数据跨境处理和数据存储区域的隐私政策表述。
4. Apple 原生崩溃诊断数据的保留说明。

## 不进入第一版的功能

1. IAP/订阅。
2. UGC 社区。
3. 医疗诊断。
4. 广告追踪。
5. 中国大陆提交。

## 美区重点

1. App Review。
2. App Privacy Label：账号、用户内容、照片/视频、健康或成长记录。
3. Account deletion。
4. COPPA 风险判断：App 面向父母，不直接面向儿童。
5. CCPA/CPRA 风险判断。
6. 不夸大健康、治疗、医疗效果。
7. Review Notes 说明照片原图私有备份和删除路径。
8. Diagnostics / 崩溃上报按实际方案披露。

## 港区重点

1. App Review。
2. PDPO 个人资料私隐条例。
3. 英文隐私政策，必要时繁中。
4. 说明个人资料用途。
5. 说明照片原图云备份。
6. 说明是否转移给第三方或跨境处理。
7. 说明用户访问、更正、删除渠道。
8. 说明崩溃诊断数据用途和保留。
