# PRIVACY_SAFE_TRACKING.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：隐私安全指标策略初版
- 日期：2026-05-28
- 当前结论：第一版默认不接入第三方分析 SDK

## 已确认事实

1. 第一版免费验证，目标是连续记录 30 天。
2. 产品处理儿童、照片、成长记录、疫苗等高敏感数据。
3. 第一版需要账号、备份恢复和照片原图云备份。
4. 第一版不做广告和商业化。
5. 当前没有批准任何第三方分析 SDK。
6. 崩溃上报使用 Apple 原生渠道。

## 合理推断

1. 产品确实需要知道留存和核心路径是否跑通，但不能采集宝宝内容。
2. 第一版可以先使用本地测试、TestFlight 访谈和手工反馈验证。
3. 如果后续接入指标，应只采集行为事件，不采集内容 payload。

## 待我确认的问题

1. 第一版是否需要任何线上埋点。
2. 是否允许使用 Apple 自带 App Analytics / TestFlight 反馈作为主要数据来源。
3. 是否允许使用 Apple 原生崩溃报告数据作为稳定性指标。
4. 账号和照片备份相关错误是否只做本地诊断日志。

## 不进入第一版的功能

1. 第三方广告 SDK。
2. 第三方归因 SDK。
3. 热力图 SDK。
4. 记录内容埋点。
5. 宝宝照片或备注内容分析。
6. 照片上传内容、文件名或对象 key 埋点。

## 可以考虑的隐私安全指标

| 指标 | 是否可采集 | 说明 |
|---|---|---|
| 首次创建宝宝档案是否完成 | 待确认 | 不上传宝宝姓名、生日 |
| 每日是否有记录行为 | 待确认 | 只计数，不上传内容 |
| 连续记录天数 | 待确认 | 可本地计算，上传需确认 |
| 月报是否打开 | 待确认 | 不上传报告内容 |
| 照片导入是否成功 | 待确认 | 不上传照片或文件名 |
| 权限拒绝率 | 待确认 | 不上传用户内容 |
| 崩溃率 | 是，Apple 原生渠道 | 需确保日志脱敏 |
| API/备份失败类别 | 待确认 | 不上传宝宝内容、照片对象 key、记录明细 |

## 禁止采集

1. 宝宝昵称、真实姓名。
2. 出生日期。
3. 照片、照片文件名、照片 EXIF。
4. 喂养、睡眠、排便、身高体重明细。
5. 疫苗名称和日期。
6. 备注内容。
7. 联系人、定位、剪贴板。

## 当前结论

第一版优先不接入第三方分析 SDK。若要验证 30 天留存，先用 TestFlight 真实家庭回访、App Store Connect 基础数据和用户主动反馈。
