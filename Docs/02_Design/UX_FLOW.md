# UX_FLOW.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：UX 流程初版
- 日期：2026-05-28

## 已确认事实

1. 第一版只做 iOS 原生 App。
2. 目标用户以妈妈为主。
3. 用户当前主要用微信记录，痛点是不好看、缺少成长档案感。
4. 第一版要服务每天多次打开、连续记录 30 天。
5. 细节字段作为备选项目，不能阻碍快速记录。
6. 第一版需要账号、备份恢复和照片原图云备份。
7. 首发地区为香港和美国。
8. 疫苗模板覆盖国内 + 香港，崩溃上报进入第一版并使用 Apple 原生渠道。

## 合理推断

1. 高频记录必须支持半夜、单手、疲惫、焦虑场景。
2. 首页应该让用户 1 步进入喂养、睡眠、排便、照片。
3. 所有表单应先给“可保存的最小记录”，再允许补充细节。
4. 空状态需要温柔，不制造“你没有好好记录”的压力。

## 首次启动流程

1. 打开 App。
2. 看到简短欢迎与隐私提示：数据默认本地优先，账号备份会上传记录和照片原图。
3. 创建宝宝档案：昵称、出生日期为核心字段；性别、头像可选。
4. 进入账号/备份引导或今日首页，具体时机待确认。

### 空状态

- 未创建宝宝档案：提示“先为宝宝建一个小档案”，提供创建按钮。
- 已创建档案但无记录：提示“今天还没有记录，先记一件小事吧”，显示快速记录入口。

## 今日首页流程

1. 显示宝宝昵称、出生第几天、纪念日提示。
2. 显示今日摘要：喂养、睡眠、排便、照片。
3. 显示快速记录按钮：喂养、睡眠、排便、照片。
4. 下方展示最近记录时间线。

### 疲劳场景检查

- 高频按钮必须可单手点击。
- 不应要求用户在保存前填写大量字段。
- 默认使用当前时间。
- 用户可以之后编辑细节。

## 喂养记录流程

1. 点击“喂养”。
2. 默认时间为当前时间。
3. 选择类型：母乳、瓶喂、配方奶、辅食。
4. 可选填写数量、时长、备注。
5. 保存并返回首页。

### 空状态

- 今日无喂养记录时显示柔和提示和新增入口。

### 错误状态

- 数量格式错误时提示用户修正。
- 保存失败时保留表单内容，不丢失输入。

## 睡眠记录流程

1. 首页点击“开始睡眠”。
2. 首页显示进行中的睡眠状态。
3. 宝宝醒来时点击“结束睡眠”。
4. 可进入编辑页修正开始/结束时间。

### 错误状态

- 结束时间早于开始时间时不能保存。
- 如果存在进行中睡眠，再次开始时提示处理上一段睡眠。

## 排便记录流程

1. 点击“排便”。
2. 默认只需记录当前时间。
3. 可选选择颜色、形态、备注。
4. 保存并返回首页。

### 错误状态

- 可选项缺失不算错误。
- 保存失败时保留输入内容。

## 照片时间线流程

1. 点击“照片”或进入时间线。
2. 选择拍照或从相册选择。
3. 系统权限弹窗由 iOS 触发。
4. 选择照片后复制到 App 私有空间。
5. 用户可设置日期和备注。
6. 照片出现在宝宝时间线。

### 权限状态

- 相册权限拒绝：显示去系统设置的说明。
- 相机权限拒绝：显示替代入口“从相册选择”。

## 疫苗提醒流程

1. 进入疫苗提醒。
2. 选择国内模板、香港模板或手动新增。
3. 模板生成提醒后，用户可以编辑名称、日期、备注。
4. 到期前通过本地通知提醒。
5. 用户可标记完成。

### 边界

- 第一版不提供医疗建议。
- 国内 + 香港模板只作为提醒模板，不提供医疗建议。

## 月度成长报告流程

1. 进入“月报”。
2. 默认显示最近一个月。
3. 聚合该月照片、喂养、睡眠、排便、身高体重、纪念日。
4. 记录不足时显示温柔空状态。

### 边界

- 不伪造不存在的数据。
- 不把报告写成医学判断。

## 数据删除流程

1. 用户进入设置或档案管理。
2. 选择删除单条记录、单张照片或宝宝档案。
3. 高风险删除需要二次确认。
4. 删除照片必须说明本地和云端原图删除结果。
5. 删除账号必须说明账号、记录、云端照片和备份数据的删除结果。

## 断网流程

1. 所有记录先写入本地。
2. 无网络时不阻止记录。
3. 断网期间显示待备份状态。
4. 恢复网络后再同步。
