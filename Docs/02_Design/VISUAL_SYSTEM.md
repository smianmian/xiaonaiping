# VISUAL_SYSTEM.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：视觉系统初版
- 日期：2026-05-28

## 已确认事实

1. 产品需要更温柔、有纪念册感。
2. 当前替代方案微信“不好看”。
3. 第一版只做 iOS 原生 App。
4. 宝宝照片是重要内容资产。

## 合理推断

1. 视觉应服务记录和回看，不能喧宾夺主。
2. 整体应该像私人家庭档案，而不是育儿社区或医疗后台。
3. 照片本身是最重要的情绪载体，界面不应过度滤镜化。

## 待我确认的问题

1. 品牌主色是否偏奶白、暖粉、浅黄，还是更克制的 Apple 式白底。
2. 是否需要品牌插画或小图标角色。
3. 深色模式是否第一版必须验收。

## 不进入第一版的功能

1. 复杂品牌动效。
2. 大量插画系统。
3. 节日皮肤。
4. 可自定义主题。

## 视觉方向

关键词：温柔、干净、低负担、纪念册、可信任、Apple 原生感。

避免：医疗警报感、社交媒体喧闹感、过度幼稚、强运营活动感、夸张糖果色。

## 颜色 Token

当前只定义方向，不锁定最终色值：

| Token | 用途 | 方向 |
|---|---|---|
| background | 页面背景 | 温暖白 / 系统背景 |
| surface | 卡片和区块 | 轻微暖色，不抢照片 |
| primary | 高频主操作 | 奶白系或暖黄系待确认 |
| secondary | 次级强调 | 柔和浅粉或浅桃色待确认 |
| success | 完成状态 | 低饱和绿色 |
| warning | 提醒 | 柔和琥珀色 |
| destructive | 删除 | 仅使用系统红 |

## 字体 Token

1. 使用 iOS 系统字体。
2. 支持 Dynamic Type。
3. 首页标题可以更有温度，但不使用难读字体。
4. 表单和记录列表优先清晰可读。

## 间距 Token

1. 基础间距：8pt。
2. 页面边距：16pt。
3. 区块间距：20pt。
4. 高频按钮周围保留足够留白，方便单手点击。

## 圆角 Token

1. 小控件：8pt。
2. 卡片：12-16pt。
3. 大图区域：12-16pt。
4. 不使用过度胶囊化导致信息密度下降。

## 图标风格

1. 优先使用 Apple 系统符号。
2. 图标含义必须直观。
3. 高频按钮可以使用图标 + 文案。
4. 不使用不明所以的可爱图形替代核心操作。

## 动效规则

1. 动效轻、短、可关闭。
2. 不让动效阻碍记录。
3. 支持 Reduce Motion。
4. 月报和纪念日可以有轻微仪式感，但不能影响性能。

## 插画 / Mascot 规则

第一版不强制创建 mascot。若后续加入，应只用于空状态、纪念日和月报，不进入高频操作路径。

## 深色模式规则

深色模式是否第一版必须支持待确认。即使不作为首轮视觉重点，也不得写死无法适配系统颜色的方案。

## 禁止事项

1. 禁止把宝宝数据做成冰冷医疗面板。
2. 禁止用大面积强饱和颜色压过照片。
3. 禁止用焦虑文案催促记录。
4. 禁止为了装饰增加记录步骤。
5. 禁止把隐私说明藏在难找的位置。
