# UI_CONTRACT.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：UI 契约初版
- 日期：2026-05-28
- 适用平台：iOS 第一版

## 已确认事实

1. 产品需要温柔、有纪念册感。
2. 第一版只做 iOS 原生 App。
3. 高频记录包括喂养、睡眠、排便。
4. 照片时间线是第一版的重要情感价值。
5. 细节字段作为备选项，不能压迫用户。
6. 第一版需要账号、备份恢复和照片原图云备份。
7. 疫苗模板覆盖国内 + 香港，崩溃上报进入第一版并使用 Apple 原生渠道。

## 合理推断

1. UI 应像私人宝宝成长档案，而不是医疗监控面板。
2. 首页需要清楚但不焦虑，避免红色警报式表达。
3. 高频按钮应大、稳定、易点。
4. 表单应使用渐进披露：先保存核心字段，再展开细节。

## 待我确认的问题

1. 品牌主色是否偏奶白/暖粉/浅黄，还是更接近 Apple 健康的白底克制风？
2. 是否需要宝宝头像作为首页强视觉中心？
3. 是否支持深色模式作为第一版验收项？
4. 账号登录是在首次启动强制出现，还是在开启备份时出现？
5. 备份说明是否作为设置页固定入口展示？

## 不进入第一版的功能

1. macOS 页面契约。
2. 家庭协作页面。
3. 社区或分享页面。
4. 付费页面。

## 组件规则

### Button

- 高频操作使用大号按钮，最小点击区域不低于 44pt。
- 高频按钮文案必须明确：喂养、开始睡眠、结束睡眠、排便、添加照片。
- 删除使用破坏性样式，并需要二次确认。
- 不使用“确定”“提交”这类模糊按钮，除非上下文已经非常明确。

### Card

- 卡片用于今日摘要、最近记录、报告区块。
- 卡片圆角保持克制，不做夸张拟物。
- 卡片内容必须可扫读，避免长段说明。

### Form

- 表单默认使用当前时间。
- 必填字段少于等于 2 个。
- 可选字段折叠或放在次级区域。
- 保存失败时保留用户输入。

### List

- 时间线和记录列表按时间倒序。
- 每条记录显示类型、时间、关键值。
- 左滑删除需要二次确认或撤销机制。

### Modal / Sheet

- 高频新增可使用 Sheet。
- Sheet 内容不应过长。
- 取消前如有未保存输入，需要确认。

### Empty State

- 文案温柔、低压力。
- 必须提供下一步操作。
- 禁止责备式表达。

### Error State

- 说明发生了什么。
- 给出用户能做的下一步。
- 不暴露内部错误码给普通用户。

## iOS 规则

- 使用系统导航、Tab、Sheet、DatePicker、Photo Picker、Notification Permission 等原生模式。
- 支持小屏 iPhone 的基本可用性。
- 支持动态字体的基本可读性。
- 重要操作不依赖长按或隐藏手势。
- 不在第一版使用复杂自定义控件替代系统控件。

## 视觉语言

- 关键词：温柔、干净、纪念册、低负担、可信任。
- 避免：医疗警报感、社交媒体喧闹感、过度儿童化、强运营感。
- 图片区域要尊重照片本身，不用重滤镜破坏宝宝照片。

## 隐私 UI 要求

- 首次涉及照片、通知、服务器存储时必须有清楚说明。
- 首次涉及崩溃上报时，隐私政策必须说明诊断用途。
- 设置页必须有“数据与隐私”入口。
- 服务器存储已进入第一版，必须显示同步/上传状态、照片原图云备份说明和删除说明。

## 禁止事项

- 禁止每个页面自行发明样式。
- 禁止按钮文案模糊。
- 禁止没有空状态。
- 禁止没有错误状态。
- 禁止把记录不足表达成用户失败。
- 禁止未经明确同意上传照片或宝宝数据。
