# ACCOUNT_DELETION_PLAN.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：账号删除方案更新
- 日期：2026-05-28
- 当前结论：第一版需要账号，因此账号删除是上线阻断项

## 已确认事实

1. 第一版先做单人记录。
2. 第一版需要账号。
3. 第一版需要备份恢复。
4. 服务器需要存储宝宝照片原图。
5. 有账号就必须提供账号删除入口。

## 合理推断

1. 正式上架前若切到 Sign in with Apple，需要处理 token/账号解绑和云端数据删除。
2. 删除账号必须覆盖宝宝档案、记录、提醒、照片原图和对象存储资产。
3. 删除失败必须可重试，并给用户明确状态。

## 待我确认的问题

1. 删除账号是否立即删除云端数据。
2. 是否存在短期恢复窗口。
3. 删除处理 SLA。
4. 付费 Apple Developer 账号开通时间。

## 不进入第一版的功能

1. 客服代删作为唯一方式。
2. 邮件人工删除作为唯一方式。
3. 复杂账号安全中心。

## 推荐路径

设置 -> 账号与安全 -> 删除账号 -> 说明后果 -> 二次确认 -> 删除账号与云端备份

## 删除范围

1. 账号资料。
2. 宝宝档案。
3. 喂养、睡眠、排便、身高体重记录。
4. 疫苗提醒。
5. 云端照片原图和缩略图，如有。
6. 本地数据，需用户确认。
7. 备份数据和日志，按保留规则说明。
8. Apple 登录 token 撤销，如使用 Sign in with Apple。

## App Store 审核要求

1. App 内必须能找到删除账号入口。
2. Review Notes 必须说明删除路径。
3. 删除路径不能只要求联系客服。
4. 删除后云端照片和记录的处理时间必须在隐私政策中说明。
