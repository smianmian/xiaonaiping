- 阶段：公开仓库的云端部署原则
- 日期：2026-05-28
- 当前结论：第一版需要后端规划；具体部署参数不得写入公开仓库

## 已确认事实

1. 第一版需要账号、备份恢复和照片原图云备份。
2. 小奶瓶涉及儿童、照片、家庭成长记录，属于高敏感数据场景。
3. 首发地区为香港和美国，中国大陆不进入第一版首发。
4. 如与其他服务同机部署，必须做到目录、端口、进程、反向代理和日志完全隔离。

## 公开仓库只保留的原则

1. 不写真实服务器地址、运维入口、登录方式、端口、目录、进程名、对象存储桶名或密钥。
2. 不固定公开云厂商、服务器区域或对象存储区域。
3. 不提供可直接执行的生产部署脚本。
4. 不把生产 `.env`、数据库连接串、云厂商 AK/SK、SSH Key 或证书放入仓库。
5. 不让客户端直接持有对象存储写入凭证。

## 同机隔离原则

如果部署到共享服务器：

1. 小奶瓶必须使用独立部署目录。
2. 小奶瓶必须使用独立内部端口。
3. 小奶瓶必须使用独立进程名。
4. 小奶瓶必须使用独立反向代理站点和 API 域名。
5. 禁止复用、覆盖、迁移或重启同机已有服务。
6. 所有部署命令必须只作用于小奶瓶自己的目录和进程。

## 安全底线

1. 公开仓库不保存生产基础设施细节。
2. 运维面板不暴露给无关公网访问。
3. 数据库不直接开放公网访问。
4. 日志不得记录儿童、照片、家庭、账号或健康相关敏感内容。
5. 删除机制、备份恢复和对象存储生命周期必须在发布前完成隐私审查。
