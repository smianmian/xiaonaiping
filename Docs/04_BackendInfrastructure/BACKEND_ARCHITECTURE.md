# BACKEND_ARCHITECTURE.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：后端架构更新
- 日期：2026-05-28
- 当前结论：第一版后端只做账号、备份恢复、照片原图文件存储和删除机制

## 已确认事实

1. 第一版需要账号。
2. 第一版需要备份恢复。
3. 第一版服务器需要存储照片原图。
4. 第一版单人记录，不做家庭协作。
5. 第一版不做管理后台。
6. 数据高敏感。
7. API 服务与对象存储区域需在发布前按合规策略确认。

## 合理推断

1. 后端应只提供备份/恢复 API 和文件存储，不做运营查询。
2. 不应在第一版做客服随意查看用户宝宝数据的后台。
3. 服务端日志必须避免敏感明文。
4. 照片应使用私有对象存储，不应公开可访问。

## 待我确认的问题

1. 是否保存照片缩略图。
2. 删除 SLA 和数据保留周期。
3. API 正式域名。
4. 对象存储私有桶命名与访问控制细节。

## 不进入第一版的功能

1. 管理后台。
2. BI 数据仓库。
3. 多人协作服务。
4. 推送运营系统。
5. 中国大陆首发服务。

## 最小后端边界

1. 账号/身份：开发期先用当前可用账号/开发者资源；正式上架前切到付费 Apple Developer 账号支持的正式能力。
2. 备份 API：上传和恢复宝宝档案、记录、提醒、照片元数据。
3. 文件 API：上传、下载、删除用户主动加入 App 的照片原图。
4. 删除 API：删除账号、记录、照片对象和云端备份。
5. 同步状态 API：返回对象版本、上传状态和删除状态。

## 安全边界

1. 所有接口必须鉴权。
2. 不允许通过可猜 ID 访问数据。
3. 不允许服务端日志记录宝宝姓名、生日、备注、照片 URL 明文。
4. 不允许客服或后台随意查看宝宝数据。
5. 不允许把敏感数据交给第三方分析。
6. 对象存储必须私有访问。
