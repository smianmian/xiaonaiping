# SAME_HOST_ISOLATION_HANDOFF.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：公开仓库的同机部署隔离交接
- 日期：2026-05-28
- 目标：说明隔离原则，不暴露生产服务器细节

## 公开仓库不记录

1. 真实服务器 IP 或主机名。
2. 运维面板入口。
3. 现有服务目录、端口、进程名或域名。
4. 小奶瓶正式部署目录、端口、进程名或域名。
5. 数据库地址、账号、密码或连接串。
6. 对象存储桶名、AK/SK、访问策略或 CDN 域名。

## 已确认原则

1. 小奶瓶后端如果与其他服务同机部署，必须完全隔离。
2. 小奶瓶只能操作自己的部署目录、进程、日志和反向代理站点。
3. 部署脚本必须排除 `.env`，不能覆盖服务器端生产配置。
4. 禁止执行会影响同机其他服务的全量重启、全量清理或全量迁移命令。
5. 上线前必须完成账号删除、数据删除、日志脱敏、备份恢复和对象存储生命周期验证。

## 私有部署文档必须包含

| 项 | 要求 |
|---|---|
| 小奶瓶部署目录 | 独立且只属于小奶瓶 |
| 小奶瓶内部端口 | 独立且不与其他服务冲突 |
| 小奶瓶进程名 | 独立且可单独 reload |
| 小奶瓶 API 域名 | 独立反向代理站点 |
| 现有服务保护清单 | 仅保存在私有运维文档 |
| 回滚命令 | 只影响小奶瓶进程和目录 |
| 环境变量 | 只保存在服务器或密钥管理系统 |

## 部署前核对

1. 确认目标路径不是任何现有服务目录。
2. 确认内部端口未被占用。
3. 确认进程管理命令只作用于小奶瓶。
4. 确认反向代理站点只转发小奶瓶 API 域名。
5. 确认日志不包含宝宝内容、照片对象 key、账号 token 或服务器密钥。
6. 确认备份和删除流程可以独立验证。

## 不允许

1. 在公开仓库写真实部署参数。
2. 在公开仓库提交生产 `.env` 或证书。
3. 使用全量重启命令作为部署步骤。
4. 让客户端直接访问对象存储写入凭证。
5. 在没有隐私审查前接入照片上传、账号、云备份或后端存储。
