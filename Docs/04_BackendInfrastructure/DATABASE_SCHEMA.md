# DATABASE_SCHEMA.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：数据库设计更新
- 日期：2026-05-28
- 当前结论：需要服务端数据库保存账号和备份元数据；照片原图放对象存储

## 已确认事实

1. 本地数据库是第一来源。
2. 第一版需要账号。
3. 第一版需要备份恢复。
4. 服务器需要存照片原图。
5. 数据包含儿童和家庭敏感信息。

## 合理推断

1. 服务端数据库应保存最小备份字段和照片对象元数据。
2. 照片二进制不应进入关系数据库，应进入对象存储。
3. 月报可由客户端生成，不必保存冗余内容。
4. 删除字段和保留周期必须设计在 schema 里。

## 待我确认的问题

1. 使用 MySQL 还是 PostgreSQL。
2. 是否保存照片缩略图对象。
3. 删除后是否有短期恢复窗口。

## 不进入第一版的功能

1. BI 宽表。
2. 用户画像表。
3. 广告归因表。
4. 内容审核队列表。

## 数据库安全要求

1. 所有表必须按 userId 隔离。
2. 高敏感字段不得写入非必要索引。
3. 备份文件必须有访问控制。
4. 日志和错误上报不得包含敏感字段。
5. 照片对象 key 不得包含宝宝姓名、生日或原始文件名。
