# SERVER_MONITORING.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：服务监控规划更新
- 日期：2026-05-28
- 当前结论：第一版需要后端，因此需要基础监控；未实施

## 已确认事实

1. 第一版需要账号、备份恢复和照片原图云备份。
2. 如果后端存在，必须有基础监控和日志。
3. 日志不得泄露宝宝敏感数据。
4. 崩溃上报使用 Apple 原生渠道。

## 合理推断

1. 监控应看服务健康，不看用户内容。
2. 崩溃或错误日志需要脱敏。
3. 照片上传/删除失败率需要监控，但不能记录照片内容或对象 key 明文。

## 待我确认的问题

1. 日志保留周期。
2. 告警渠道。
3. Apple 原生崩溃上报脱敏验证方式。

## 不进入第一版的功能

1. BI 仪表盘。
2. 用户内容查看后台。
3. 复杂 APM。

## 最小监控候选

1. API 可用性。
2. CPU / 内存 / 磁盘。
3. 数据库连接。
4. 对象存储上传/下载/删除成功率。
5. 备份是否成功。
6. 错误数量。

## 日志禁止字段

1. 宝宝昵称。
2. 出生日期。
3. 照片 URL 或对象 key 明文。
4. 喂养、睡眠、排便、身高体重明细。
5. 疫苗备注。
