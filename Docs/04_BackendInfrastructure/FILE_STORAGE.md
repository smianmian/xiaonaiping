# FILE_STORAGE.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：文件存储更新
- 日期：2026-05-28
- 当前结论：第一版需要服务器存储宝宝照片原图；只允许上传用户主动加入 App 的照片

## 已确认事实

1. 照片可以复制进 App 私有空间。
2. 服务器需要存储宝宝照片原图。
3. 第一版需要账号和备份恢复。
4. 宝宝照片属于高敏感家庭数据。
5. 首发地区为香港和美国。
6. 对象存储第一版默认按合规策略确认。

## 合理推断

1. 云端照片原图应使用私有对象存储，默认候选为对象存储。
2. 照片原图上传必须用户知情，不能自动上传系统相册。
3. 需要同时管理本地文件、缩略图、云端对象和删除状态。
4. 照片不应用于分析、训练、推荐或运营。

## 待我确认的问题

1. 是否生成和上传缩略图。
2. 是否压缩原图或保留原始质量。
3. 云端照片删除 SLA。
4. 是否需要导出月报 PDF 或图片。

## 不进入第一版的功能

1. 第三方云相册。
2. 公开照片分享。
3. 照片 AI 识别。
4. 自动上传系统相册全部照片。
5. 中国大陆对象存储首发。

## 本地文件规则

1. 用户主动选择或拍摄的照片复制进 App 私有空间。
2. 删除 App 内照片时，同时删除本地原图和缩略图。
3. 不保存外部相册永久引用作为唯一来源。
4. 本地文件删除失败必须可重试或提示。

## 云端文件规则

默认候选：对象存储 或同等私有对象存储。

必须明确：

1. 存储区域：发布前按合规策略确认。
2. 私有桶 / 私有访问策略。
3. 文件命名规则，不能包含宝宝姓名或生日。
4. 上传鉴权方式。
5. 是否生成缩略图。
6. 删除机制。
7. 保留周期。
8. 用户端明确同意和状态展示。

## 云端照片删除要求

1. 删除单张照片时删除云端原图和缩略图。
2. 删除宝宝档案时删除该宝宝全部云端照片。
3. 删除账号时删除账号下全部云端照片。
4. 删除失败必须进入重试队列。
5. 删除 SLA 必须写入隐私政策。
