## 已确认事实

1. 第一版需要账号、备份恢复和照片原图云备份。
2. 默认基线为云端后端 + 私有运维流程。
3. 首发地区为香港和美国。
4. 不允许写生产密钥或服务器登录信息。
5. 如果与其他服务同机，必须使用独立目录、端口、进程和反向代理站点。
6. API 服务器和对象存储区域必须在发布前按隐私和区域合规确认。

## 合理推断

1. 服务器和对象存储区域必须匹配首发地区、隐私政策和 App Store 隐私披露。
2. 中国大陆服务器会触发 APP 备案 / ICP 判断，不适合当前首发默认路径。
3. 隐私政策 URL 和 API 域名也要规划。

## 同机部署边界

公开仓库只保留原则，不记录真实服务器、目录、端口、进程名、域名或运维面板入口。

如果小奶瓶与其他服务同机：

- 小奶瓶必须有独立部署目录、内部端口、进程名、日志路径和反向代理站点。
- 部署命令必须只作用于小奶瓶自己的目录和进程。
- 禁止复用、覆盖、迁移、重启或清理同机已有服务。
- 具体部署参数只允许保存在私有运维文档或服务器环境配置中。

## 禁止事项

1. 不在公开文档里写真实 IP、密码、AK/SK、运维入口、端口、目录或域名。
2. 不在未备案情况下假定中国大陆首发。
3. 不用可能影响同机其他服务的全量命令作为交付指令。
