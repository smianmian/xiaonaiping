# API_DESIGN.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：API 设计更新
- 日期：2026-05-28
- 当前结论：需要账号、备份恢复、照片原图上传/下载/删除 API；未实现

## 已确认事实

1. 第一版必须离线可本地记录。
2. 第一版需要账号。
3. 第一版需要备份恢复。
4. 服务器需要存照片原图。
5. 首发地区为香港和美国。

## 合理推断

1. API 不应参与高频记录的实时保存路径。
2. API 失败不能影响本地使用。
3. 删除 API 是上线前强制项。
4. 照片上传应只处理用户主动加入 App 的照片。

## 待我确认的问题

1. 照片上传是否使用预签名 URL。
2. 是否上传缩略图。
3. 删除 SLA。
4. API 正式域名。

## 不进入第一版的功能

1. 管理后台 API。
2. 运营推送 API。
3. 家庭协作 API。
4. 公开分享 API。

## 必需 API，设计待细化

| API | 用途 | 第一版状态 |
|---|---|---|
| Auth | 用户身份 | 必需 |
| Backup Upload | 上传本地记录备份 | 必需 |
| Backup Restore | 恢复用户记录 | 必需 |
| Photo Upload | 上传用户主动加入 App 的照片原图 | 必需 |
| Photo Download | 恢复照片原图或缩略图 | 必需 |
| Object Delete | 删除云端照片对象 | 必需 |
| Account Delete | 删除账号和云端数据 | 必需 |

## 禁止事项

1. 不允许上传未在隐私审查中列出的字段。
2. 不允许把宝宝照片、备注、生日作为日志输出。
3. 不允许未登录或未鉴权访问用户数据。
4. 不允许返回长期公开照片 URL。
