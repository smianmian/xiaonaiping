# BACKEND_DECISION.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：AppLaunchOS v2 后端决策更新
- 日期：2026-05-28
- 当前结论：第一版需要后端能力，但只允许做最小账号 + 备份恢复 + 照片原图文件存储；未进入代码实现

## 已确认事实

1. 第一版只做 iOS 原生 App。
2. 第一版先做单人记录。
3. 第一版免费验证，先不商业化。
4. 数据本地优先。
5. 第一版需要账号。
6. 第一版需要备份恢复。
7. 服务器需要存储宝宝照片原图。
8. 首发地区为香港和美国，中国大陆不首发。
9. 数据包含儿童、照片、成长记录、疫苗提醒等高敏感家庭数据。
10. 如果后端进入实现，公开仓库只保留架构原则；具体部署参数放在私有运维文档。
11. 如果与其他服务同机部署，小奶瓶必须完全隔离。
12. 疫苗提醒模板覆盖国内 + 香港，崩溃上报进入第一版。
13. API 服务和对象存储区域必须在发布前按隐私和区域合规确认。
14. 崩溃上报使用 Apple 原生渠道。

## 合理推断

1. 后端的第一版目的应严格限定为账号下的备份/恢复。
2. 项目需要高于纯本地 Level 0，但不需要完整 Level 3 的管理后台、BI 或运营系统。
3. 最合适的描述是 Level 2.5：账号 + 云端备份恢复 + 对象存储，不做订阅、远程配置、管理后台、BI。
4. 照片原图上传会显著增加隐私、成本、对象存储、删除验证和 App Store 隐私披露压力。
5. 开发期账号先用当前可用资源，正式上架前切到付费 Apple Developer 账号支持的正式能力。
6. 同机部署必须使用独立目录、端口、进程和反向代理站点，不能影响任何已有服务。

## 基础设施等级判断

| Level | 名称 | 小奶瓶当前判断 |
|---|---|---|
| Level 0 | 纯本地客户端 | 不满足已确认的备份恢复和原图云备份 |
| Level 1 | 本地优先 + 匿名指标 | 不满足账号和备份恢复 |
| Level 2 | 账号 + 订阅 + 远程配置 | 账号需要，但订阅和远程配置不需要 |
| Level 3 | 云同步 + 管理后台 + BI | 需要云端备份，但不需要管理后台和 BI |
| Level 4 | SaaS / 多租户 / 企业权限 | 不适用 |

当前选择：**Level 2.5：本地优先 + 账号 + 备份恢复 + 照片原图对象存储**。

## 同机隔离决策

如果与其他服务同机，必须保持已有服务不受影响：

- 不复用或覆盖已有服务目录。
- 不复用已有服务端口。
- 不执行会影响其他服务的全量重启、全量清理或全量迁移命令。

小奶瓶部署参数必须在私有运维文档中单独确认：

- 独立部署目录。
- 独立内部端口。
- 独立进程名。
- 独立反向代理站点和 API 域名。
