# AUTH_AND_ACCOUNT.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：账号方案更新
- 日期：2026-05-28
- 当前结论：第一版需要账号；开发期先用当前可用账号/开发者资源，正式上架前切到付费 Apple Developer 账号支持的正式能力

## 已确认事实

1. 第一版先做单人记录。
2. 第一版需要备份恢复。
3. 第一版需要服务器存储照片原图。
4. 第一版需要稳定账号身份。
5. 首发地区为香港和美国。
6. 开发期账号先用当前可用方案；后续开通付费 Apple Developer 账号。

## 合理推断

1. Sign in with Apple 是正式 App Store 上架前最稳妥的账号方向。
2. 手机号/邮箱会增加隐私、验证码、客服和合规成本。
3. 有账号就必须有账号删除。
4. 账号不应变成家庭协作或社交身份系统。

## 待我确认的问题

1. 付费 Apple Developer 账号开通时间。
2. 是否允许邮箱登录作为补充。
3. 删除账号后云端数据是否立即硬删除，还是短期保留恢复窗口。

## 不进入第一版的功能

1. 手机号注册。
2. 微信登录。
3. Google / Facebook 等第三方社交登录。
4. 家庭成员邀请。
5. 权限角色系统。
6. 账号公开主页。

## 推荐账号方案

第一版推荐：

1. 开发期：先使用当前可用账号/开发者资源完成账号流程规格和测试。
2. 正式上架前：开通付费 Apple Developer 账号，完成 Sign in with Apple、TestFlight、App Store Connect 和隐私资料。
3. 本地数据可以先创建，用户在开启备份恢复时再登录。
4. 登录后将本地数据绑定到账号并进入备份队列。
5. 退出登录不删除本地数据，删除账号才触发云端删除流程。

## 账号删除要求

账号进入第一版，必须同步完成：

1. `Docs/05_BusinessOperations/ACCOUNT_DELETION_PLAN.md`
2. App 内删除账号入口。
3. 云端宝宝档案、记录、照片原图删除机制。
4. Apple 登录 token 撤销，如使用 Sign in with Apple。
5. App Store Review Notes 删除路径说明。
