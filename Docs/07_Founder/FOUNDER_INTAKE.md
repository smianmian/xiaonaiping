# 创始人想法输入表

## 已确认事实

1. 项目暂名：小奶瓶 / 宝宝成长记录。
2. 第一版只做 iOS 原生 App。
3. 目标用户以妈妈为主。
4. 覆盖宝宝年龄：0-3 岁。
5. 用户现在主要用微信记录。
6. 当前方案核心痛点：不好看，不像一个温柔、有纪念感的宝宝成长档案。
7. 第一版免费验证，先不商业化。
8. 产品感觉：更温柔、有纪念册感。
9. 记录方向包括宝宝档案、纪念日、喂养、睡眠、排便、身高体重、宝宝照片时间线、疫苗提醒、月度成长报告。
10. 喂养细分、辅食、睡眠开始结束、排便颜色/形态/备注等作为可选项目。
11. 第一版先做单人记录。
12. 第一版需要账号。
13. 第一版需要备份恢复。
14. 服务器需要存储宝宝照片原图。
15. 首发地区为香港和美国。
16. 照片可以复制进 App 私有空间。
17. 疫苗提醒模板覆盖国内 + 香港。
18. 崩溃上报进入第一版。
19. 账号开发期先用当前可用账号/开发者资源，正式上架前开通付费 Apple Developer 账号。
20. API 服务和对象存储区域必须在发布前按隐私和区域合规确认。
21. 崩溃上报使用 Apple 原生渠道。
22. macOS 第一版不做。

## 合理推断

1. 第一版最关键的不是功能数量，而是高频记录速度、情绪负担低、回看有价值。
2. 首页应该围绕“今天宝宝发生了什么”和快速记录，而不是复杂报表。
3. 后端服务器存储进入第一版后，应定位为第一方账号备份/恢复能力，而不是社交、协作或数据分析平台。
4. 照片原图上传服务器是第一版高风险能力，必须有明确告知、删除机制和对象存储安全设计。
