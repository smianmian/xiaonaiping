# Decisions

## 2026-05-28

### 已确认事实

- 第一版只做 iOS 原生 App。
- macOS、Android、Web 不进入第一版。
- 目标用户以妈妈为主，覆盖宝宝 0-3 岁。
- 第一版免费验证，不做商业化。
- 记录数据本地优先，同时需要评估第一方后端服务器存储。
- 照片可复制进 App 私有空间。
- 首发地区：香港和美国。
- 第一版需要账号。
- 第一版需要备份恢复。
- 服务器需要存储宝宝照片原图。
- 新框架默认后端部署基线为公开仓库部署原则，但不代表第一版必须做后端。
- 疫苗提醒模板范围：国内 + 香港。
- 崩溃上报进入第一版，使用 Apple 原生渠道。
- 账号策略：开发期先用当前可用账号/开发者资源推进；正式上架前必须准备付费 Apple Developer 账号并完成对应能力。
- 同机部署必须与同机已有服务完全隔离，不能复用已有服务资源或执行全量影响命令。
- API 服务和对象存储区域必须在发布前按隐私和区域合规确认；这里的“区域”就是服务器和照片仓库所在的数据中心位置。

### 合理推断

- 服务器存储应保持最小化，服务账号下的备份/恢复，而不是社交或分析。
- 第一版视觉应温柔、干净、有纪念册感。
- 首发香港和美国意味着中国大陆不进入首发；中国大陆后续上线需要提前处理 APP 备案、ICP、儿童/个人信息保护和 SDK 披露。
- 服务器存照片原图会显著提高隐私、安全、存储成本、删除验证和 App Store 隐私披露要求。
- 崩溃上报应优先不携带宝宝内容、照片对象 key、记录明细或账号可识别信息。

### 待我确认的问题

- 是否需要除 Apple 原生崩溃上报以外的线上埋点。
- 云端数据保留周期和删除 SLA。
- 小奶瓶 API 正式域名。
- 付费 Apple Developer 账号开通时间。
- Apple 原生崩溃上报的脱敏验证方式。

### 不进入第一版的功能

- macOS、Android、Web、家庭协作、社区、电商、广告、医疗诊断。
- IAP、订阅、付费墙。
- 管理后台、BI、复杂运营系统。
- 中国大陆首发。
