## 已确认事实

1. 第一版只做 iOS 原生 App。
2. 目标用户以妈妈为主，覆盖宝宝 0-3 岁。
3. 用户当前主要用微信记录宝宝日常。
4. 当前方案的明确痛点是“不好看”，缺少温柔、有纪念册感的成长记录体验。
5. 第一版目标不是大而全，而是让新手妈妈每天愿意打开多次，持续记录 30 天。
6. 第一版功能方向包括宝宝档案、纪念日、喂养、睡眠、排便、身高体重、照片时间线、疫苗提醒、月度成长报告。
7. 喂养类型、辅食、睡眠开始结束、排便颜色/形态/备注等细节作为备选字段。
8. 第一版先做单人记录。
9. 第一版需要账号。
10. 第一版需要备份恢复。
11. 服务器需要存储宝宝照片原图。
12. 首发地区为香港和美国。
13. 照片可以复制进 App 私有空间。
14. 疫苗提醒模板覆盖国内 + 香港。
15. 崩溃上报进入第一版，使用 Apple 原生渠道。
16. API 服务与对象存储区域需在发布前按合规策略确认。
17. macOS 第一版不做。
18. 第一版免费验证，先不商业化。

## 合理推断

1. 小奶瓶的核心竞争力应来自“记录低负担 + 回看有情感价值”，而不是功能堆叠。
2. iOS 首页应优先承载今日状态和快速记录入口。
3. 后端服务器存储应限定为第一方账号下的最小化备份/恢复，不应在第一版扩展成社交、分析或运营后台。
4. 照片原图上传服务器会显著增加隐私、成本和审核风险，需要在隐私政策、App Store 隐私标签、删除机制和对象存储设计中单独处理。
5. 疫苗提醒第一版覆盖国内 + 香港模板，但必须避免医疗建议表达。
6. 崩溃上报应只用于稳定性，不采集宝宝内容、照片原图、对象 key 或记录明细。

## 1. 产品一句话

小奶瓶是一款面向新手妈妈的 iOS 原生宝宝成长记录 App，用温柔、低负担、有纪念册感的方式记录宝宝 0-3 岁的喂养、睡眠、排便、照片、成长指标、疫苗提醒和月度成长报告。

## 2. 目标用户

### 核心用户

- 新手妈妈。
- 宝宝年龄 0-3 岁。
- 需要每天多次记录宝宝状态。
- 当前可能用微信、相册、备忘录或纸质方式零散记录。

### 次级用户

- 爸爸、祖辈、其他照护者暂不作为第一版核心用户。
- 第一版不做多人协作，但内容表达不能排斥非妈妈照护者。

## 3. 核心场景

1. 半夜或刚醒时快速记录一次喂养。
2. 宝宝睡着或醒来时记录睡眠开始/结束。
3. 换尿布后记录排便。
4. 定期记录身高体重。
5. 给宝宝拍照后存入成长时间线。
6. 查看出生第几天、满月、百天等纪念日。
7. 设置或查看疫苗提醒。
8. 月末生成宝宝成长月报，用于回顾和保存。

## 4. 核心问题

1. 微信记录零散，难以形成宝宝成长档案。
2. 普通聊天记录或备忘录不够好看，缺少仪式感。
3. 高频记录容易疲惫，步骤多会导致放弃。
4. 照片、出生日期、成长记录和疫苗信息敏感，不能随意上传第三方。
5. 新手妈妈需要快速知道“今天记了什么”和“宝宝最近变化如何”。

## 5. 产品解决方案

1. 用 iOS 原生体验提供低负担快速记录。
2. 用时间线把日常记录和宝宝照片组织成可回看的成长档案。
3. 用纪念日和月度成长报告增强情感价值。
4. 默认本地优先，照片复制到 App 私有空间。
5. 第一方后端服务器存储用于账号下的备份/恢复，并包含照片原图备份；实现前必须通过隐私、安全、同步和发布合规审查。

## 6. MVP 功能范围

1. 宝宝档案。
2. 首页今日概览。
3. 纪念日展示。
4. 喂养记录。
5. 睡眠记录。
6. 排便记录。
7. 身高体重记录。
8. 宝宝照片时间线。
9. 疫苗提醒。
10. 月度成长报告。
11. 本地数据删除机制。
12. 账号、备份恢复、照片原图云备份、服务器存储规则和隐私披露。
13. 崩溃上报。

## 7. 用户价值

1. 记录更快：减少新手妈妈在高压场景下的操作负担。
2. 回看更美：把零散记录变成有纪念感的成长档案。
3. 数据更安心：本地优先，敏感数据处理透明。
4. 持续更容易：用今日概览、纪念日和月报建立持续记录动机。

## 8. 商业假设

第一版免费验证，不设置商业化入口。产品验证重点是留存、记录频次和真实家庭反馈。

后续商业化可能方向需另行确认，包括高级导出、家庭协作、云备份、实体纪念册等，但均不进入第一版。

## 9. 隐私假设

1. 宝宝姓名/昵称、出生日期、照片、喂养、睡眠、排便、身高体重、疫苗提醒均属于高敏感家庭数据。
2. 本地数据应存储在 App 沙盒中。
3. 后端服务器存储必须由用户明确知情，并提供删除机制。
4. 不允许接入第三方分析、广告或不必要 SDK。

## 10. 风险假设

1. 功能太多会稀释第一版的记录体验。
2. 后端服务器存储会增加隐私、合规、账号、成本和安全复杂度。
3. 疫苗提醒若做成医学建议，可能触发审核和责任风险。
4. 照片原图上传服务器会显著增加隐私风险。
5. 如果首页不能让用户 1-2 步完成高频记录，30 天留存目标会受影响。

## 11. 验收标准摘要

1. 高频记录功能均支持新增、编辑、删除、空状态、错误状态和断网行为。
2. 所有敏感数据有明确存储位置、删除方式、同步/上传说明。
3. 用户可以在无网络情况下完成本地记录。
4. 首页能清楚展示今天已记录内容和快速记录入口。
5. 月度成长报告能基于真实本地数据生成。
6. 不接入任何未通过隐私审查的数据能力。
