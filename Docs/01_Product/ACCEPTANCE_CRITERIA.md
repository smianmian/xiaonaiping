# ACCEPTANCE_CRITERIA.md

## 文档状态

- 项目：小奶瓶 / 宝宝成长记录
- 阶段：验收标准初版
- 日期：2026-05-28

## 已确认事实

1. 第一版只验收 iOS 原生 App。
2. 第一版必须支持宝宝档案、纪念日、喂养、睡眠、排便、身高体重、照片时间线、疫苗提醒、月度成长报告。
3. 第一版单人记录。
4. 照片复制到 App 私有空间。
5. 后端服务器存储已确认用于账号下的备份恢复。
6. 服务器需要存储宝宝照片原图。
7. 首发地区为香港和美国。
8. 疫苗提醒模板覆盖国内 + 香港。
9. 崩溃上报进入第一版，使用 Apple 原生渠道。
10. API 服务与对象存储区域需在发布前按合规策略确认。

## 合理推断

1. 所有记录类功能都必须支持新增、编辑、删除、空状态、错误状态。
2. 所有高频记录都必须在断网时可用。
3. 细节字段不能阻碍快速记录。

## 待我确认的问题

1. 是否必须支持多个宝宝档案。
2. 付费 Apple Developer 账号开通时间。
3. 云端数据保留周期和删除 SLA。

## 不进入第一版的功能

1. macOS 验收。
2. 多人协作验收。
3. 社区、分享、电商、订阅验收。

## 通用验收标准

每个功能完成前必须满足：

- [ ] 可以新增。
- [ ] 可以编辑。
- [ ] 可以删除。
- [ ] 有空状态。
- [ ] 有错误状态。
- [ ] 数据保存正确。
- [ ] 断网行为明确。
- [ ] iOS 小屏显示正常。
- [ ] 支持动态字体的基本可读性。
- [ ] 涉及敏感数据时，存储位置、删除方式、上传规则明确。
- [ ] 有可执行测试用例或手工验收步骤。

## 宝宝档案

### 用户目标

用户可以创建一个宝宝档案，作为所有记录的归属。

### 完成条件

- [ ] 可填写宝宝昵称。
- [ ] 可填写出生日期。
- [ ] 可选填性别。
- [ ] 可选设置头像。
- [ ] 可编辑档案。
- [ ] 可删除档案。
- [ ] 删除档案前有二次确认。
- [ ] 删除档案时说明会影响哪些记录。
- [ ] 未创建档案时首页显示温柔空状态。

### 不接受的情况

- 未创建宝宝档案就产生无法归属的记录。
- 删除档案后仍残留不可见数据。

## 今日首页

### 用户目标

用户打开 App 后能立即看到宝宝今天状态，并快速记录。

### 完成条件

- [ ] 显示宝宝出生第几天。
- [ ] 显示今日喂养次数摘要。
- [ ] 显示今日睡眠摘要。
- [ ] 显示今日排便次数摘要。
- [ ] 提供喂养、睡眠、排便、照片的快速入口。
- [ ] 没有今日记录时显示空状态。
- [ ] 有进行中的睡眠时清楚显示。

### 不接受的情况

- 首页只做装饰，没有快速记录入口。
- 高频入口隐藏过深。

## 纪念日

### 用户目标

用户可以看到出生第几天、满月、百天等纪念节点。

### 完成条件

- [ ] 根据出生日期自动计算出生天数。
- [ ] 自动识别满月、百天等节点。
- [ ] 日期异常时显示可理解错误。
- [ ] 纪念日文案不制造焦虑。

### 不接受的情况

- 纪念日计算错误。
- 用医学或育儿结论替代纪念提示。

## 喂养记录

### 用户目标

用户可以快速记录一次喂养，并按需补充细节。

### 完成条件

- [ ] 可记录时间。
- [ ] 可选择母乳、瓶喂、配方奶、辅食。
- [ ] 数量/时长为可选字段。
- [ ] 备注为可选字段。
- [ ] 可编辑和删除。
- [ ] 今日首页能反映新增记录。
- [ ] 断网时可保存到本地。

### 不接受的情况

- 不填数量或备注就无法保存。
- 记录后首页摘要不更新。

## 睡眠记录

### 用户目标

用户可以记录睡眠开始和结束，看到睡眠时长。

### 完成条件

- [ ] 可开始一次睡眠。
- [ ] 可结束进行中的睡眠。
- [ ] 可手动填写开始/结束时间。
- [ ] 支持未结束状态。
- [ ] 可编辑和删除。
- [ ] 时间顺序错误时提示。

### 不接受的情况

- 结束时间早于开始时间仍可保存。
- 进行中状态丢失。

## 排便记录

### 用户目标

用户可以快速记录排便，并在需要时补充颜色、形态、备注。

### 完成条件

- [ ] 可记录时间。
- [ ] 颜色为可选字段。
- [ ] 形态为可选字段。
- [ ] 备注为可选字段。
- [ ] 可编辑和删除。
- [ ] 今日首页能反映排便次数。

### 不接受的情况

- 强制用户填写颜色或形态。
- 用排便信息给出医疗诊断。

## 身高体重记录

### 用户目标

用户可以记录并回看宝宝成长指标。

### 完成条件

- [ ] 可记录日期。
- [ ] 可记录体重。
- [ ] 可记录身高。
- [ ] 身高和体重至少填写一项即可保存。
- [ ] 可查看历史列表。
- [ ] 可编辑和删除。

### 不接受的情况

- 输入单位不明确。
- 删除后趋势或列表仍显示旧数据。

## 宝宝照片时间线

### 用户目标

用户可以把宝宝照片保存到 App 内，形成成长时间线。

### 完成条件

- [ ] 可从相册选择照片。
- [ ] 可通过系统相机添加照片。
- [ ] 照片复制到 App 私有空间。
- [ ] 可为照片设置日期。
- [ ] 可删除 App 内照片。
- [ ] 相册权限被拒绝时有友好提示。
- [ ] 不默认上传照片到服务器。

### 不接受的情况

- 只保存外部相册引用，导致 App 内时间线失效。
- 未经明确同意上传照片。

## 疫苗提醒

### 用户目标

用户可以记录和提醒疫苗相关事项。

### 完成条件

- [ ] 可新增疫苗提醒。
- [ ] 可编辑提醒日期。
- [ ] 可删除提醒。
- [ ] 可标记完成。
- [ ] 可查看未完成和已完成状态。
- [ ] 可选择国内模板。
- [ ] 可选择香港模板。
- [ ] 模板生成后用户可编辑日期和备注。
- [ ] 不提供医疗建议。

### 不接受的情况

- 把提醒文案写成医学诊断或强制建议。
- 用户无法修改实际接种日期。

## 崩溃上报

### 用户目标

开发者可以发现影响使用的崩溃，同时不采集宝宝隐私内容。

### 完成条件

- [ ] 崩溃上报方案写入 `SDK_DATA_INVENTORY.md`。
- [ ] 不上传宝宝照片、照片对象 key、生日、记录明细、备注。
- [ ] App Store 隐私标签与实际行为一致。
- [ ] 用户隐私政策说明崩溃数据用途。

### 不接受的情况

- 崩溃日志包含宝宝内容或照片路径明文。
- 接入第三方 SDK 但未更新隐私审查。

## 月度成长报告

### 用户目标

用户可以查看宝宝某个月的成长记录总结。

### 完成条件

- [ ] 可选择月份。
- [ ] 聚合该月喂养、睡眠、排便、身高体重、照片、纪念日。
- [ ] 记录不足时显示友好空状态。
- [ ] 报告内容来自真实记录。
- [ ] 不伪造不存在的数据。

### 不接受的情况

- 记录不足时强行生成虚假总结。
- 报告页面无法追溯到原始记录。

## 账号、备份恢复和服务器存储

### 用户目标

用户可以登录账号，备份和恢复宝宝记录与照片原图，并拥有删除能力。

### 完成条件

- [ ] 实现前完成 `SYNC_RULES.md` 和 `PRIVACY_REVIEW.md`。
- [ ] 明确上传字段。
- [ ] 明确照片原图上传范围，只允许上传用户主动加入 App 的照片。
- [ ] 明确账号或设备身份识别方式。
- [ ] 明确加密传输。
- [ ] 明确服务器删除机制。
- [ ] 断网时本地记录不受影响。
- [ ] 可恢复同一账号下的宝宝记录。
- [ ] 可删除云端照片原图。
- [ ] 有账号删除入口。

### 不接受的情况

- 未告知用户就上传宝宝数据。
- 没有删除机制。
- 使用第三方数据服务但未披露。
- 删除本地照片后云端原图仍不可控。
